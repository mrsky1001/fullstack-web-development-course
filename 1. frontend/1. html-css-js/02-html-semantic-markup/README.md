## 📚 Теоретическая часть

### Что такое семантика?

**Семантика** (от греч. "semantikos" — значащий) — это раздел лингвистики, изучающий значение слов и выражений.

В контексте HTML, **семантическая разметка** означает использование тегов, которые описывают **смысл** содержимого, а не только его внешний вид.

### Сравнение: несемантический vs семантический код

**До HTML5 (несемантический):**
```html
<div id="header">
    <div class="nav">...</div>
</div>
<div id="main">
    <div class="article">...</div>
    <div class="sidebar">...</div>
</div>
<div id="footer">...</div>
```

**HTML5 (семантический):**
```html
<header>
    <nav>...</nav>
</header>
<main>
    <article>...</article>
    <aside>...</aside>
</main>
<footer>...</footer>
```

### Почему семантика важна?

| Аспект | Описание |
|--------|----------|
| **SEO** | Поисковые системы лучше понимают структуру страницы |
| **Доступность** | Программы чтения с экрана правильно интерпретируют контент |
| **Читаемость кода** | Разработчикам проще понимать структуру |
| **Поддержка** | Легче вносить изменения |
| **Стандарты** | Соответствие современным веб-стандартам |

## 🏗 Структурные семантические теги

### Основная структура страницы

```
┌─────────────────────────────────────────────────┐
│                  <header>                        │
│  ┌─────────────────────────────────────────┐    │
│  │              <nav>                       │    │
│  └─────────────────────────────────────────┘    │
└─────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────┐
│                   <main>                         │
│  ┌───────────────────────┐ ┌───────────────┐    │
│  │      <article>        │ │   <aside>     │    │
│  │  ┌─────────────────┐  │ │               │    │
│  │  │   <section>     │  │ │               │    │
│  │  └─────────────────┘  │ │               │    │
│  │  ┌─────────────────┐  │ │               │    │
│  │  │   <section>     │  │ │               │    │
│  │  └─────────────────┘  │ │               │    │
│  └───────────────────────┘ └───────────────┘    │
└─────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────┐
│                  <footer>                        │
└─────────────────────────────────────────────────┘
```

### Описание тегов

#### `<header>` — Шапка

```html
<header>
    <img src="logo.png" alt="Логотип">
    <h1>Название сайта</h1>
    <nav>...</nav>
</header>
```

**Применение:**
- Шапка страницы (логотип, меню)
- Шапка статьи (заголовок, дата, автор)
- Шапка секции

> **Важно:** На странице может быть несколько `<header>`, но обычно один в шапке сайта.

#### `<nav>` — Навигация

```html

<nav>
    <ul>
        <li><a href="/">Главная</a></li>
        <li><a href="/about">О нас</a></li>
        <li><a href="/contacts">Контакты</a></li>
    </ul>
</nav>
```

**Применение:**
- Главное меню сайта
- Меню в подвале
- Хлебные крошки
- Постраничная навигация

> **Важно:** Не каждая группа ссылок — это `<nav>`. Используйте для основной навигации.

#### `<main>` — Основной контент

```html
<main>
    <h1>Заголовок страницы</h1>
    <article>...</article>
</main>
```

**Применение:**
- Уникальный контент страницы
- Основная область, отличающая эту страницу от других

> **Важно:** Только ОДИН `<main>` на странице! Не включайте шапку, подвал, боковую панель.

#### `<article>` — Независимая статья

```html
<article>
    <header>
        <h2>Заголовок статьи</h2>
        <time datetime="2024-01-15">15 января 2024</time>
    </header>
    <p>Содержание статьи...</p>
    <footer>
        <p>Автор: Иван Иванов</p>
    </footer>
</article>
```

**Применение:**
- Блог-пост
- Новость
- Комментарий
- Товар в каталоге
- Виджет

> **Критерий:** Если контент понятен вне контекста страницы — это `<article>`.

#### `<section>` — Тематический раздел

```html
<section>
    <h2>Наши услуги</h2>
    <p>Описание услуг...</p>
</section>
```

**Применение:**
- Главы документа
- Разделы на лендинге
- Тематические группы контента

> **Важно:** `<section>` должен иметь заголовок (h2-h6). Если заголовок не нужен — используйте `<div>`.

#### `<aside>` — Побочный контент

```html
<aside>
    <h3>Реклама</h3>
    <p>Баннер...</p>
</aside>

<aside>
    <h3>Популярные статьи</h3>
    <ul>...</ul>
</aside>
```

**Применение:**
- Боковая панель (sidebar)
- Рекламные блоки
- Связанные ссылки
- Выноски (примечания)

#### `<footer>` — Подвал

```html
<footer>
    <p>&copy; 2024 Компания</p>
    <nav>
        <a href="/privacy">Политика конфиденциальности</a>
        <a href="/terms">Условия использования</a>
    </nav>
</footer>
```

**Применение:**
- Подвал сайта
- Подвал статьи (автор, теги)
- Подвал секции

## 📝 Текстовые семантические теги

### Группировка контента

| Тег | Назначение | Пример |
|-----|------------|--------|
| `<figure>` | Иллюстрация с подписью | Изображение, диаграмма, код |
| `<figcaption>` | Подпись к figure | "Рис. 1: График продаж" |
| `<blockquote>` | Длинная цитата | Цитата из книги |
| `<q>` | Короткая инлайн-цитата | Слово или фраза |
| `<cite>` | Источник цитаты | Название книги, автор |
| `<address>` | Контактная информация | Email, адрес автора |

### Примеры использования

```html
<!-- Рисунок с подписью -->
<figure>
    <img src="chart.png" alt="График продаж">
    <figcaption>Рис. 1: Динамика продаж за 2024 год</figcaption>
</figure>

<!-- Цитата -->
<blockquote cite="https://example.com/book">
    <p>Единственный способ делать великие дела — любить то, что ты делаешь.</p>
    <footer>— <cite>Стив Джобс</cite></footer>
</blockquote>

<!-- Контактная информация -->
<address>
    Автор: <a href="mailto:author@example.com">Иван Иванов</a><br>
    Москва, Россия
</address>
```

## ⏰ Время и даты

### Тег `<time>`

```html
<!-- Дата публикации -->
<time datetime="2024-01-15">15 января 2024</time>

<!-- Дата и время -->
<time datetime="2024-01-15T14:30:00+03:00">15 января в 14:30 (МСК)</time>

<!-- Продолжительность -->
<time datetime="PT2H30M">2 часа 30 минут</time>
```

**Форматы datetime:**
- `YYYY-MM-DD` — дата
- `HH:MM:SS` — время
- `YYYY-MM-DDTHH:MM:SS` — дата и время
- `PTXHXM` — продолжительность (ISO 8601)

## ⌨️ Разметка кода

### Теги для кода

```html
<!-- Инлайн-код -->
<p>Используйте функцию <code>console.log()</code> для отладки.</p>

<!-- Блок кода -->
<pre><code>
function greet(name) {
    return `Привет, ${name}!`;
}
</code></pre>

<!-- Ввод пользователя -->
<p>Нажмите <kbd>Ctrl</kbd> + <kbd>S</kbd> для сохранения.</p>

<!-- Вывод программы -->
<p>Результат: <samp>Hello, World!</samp></p>

<!-- Переменная -->
<p>Переменная <var>x</var> равна 10.</p>
```

## ♿ Доступность (Accessibility)

### Атрибуты ARIA

**ARIA** (Accessible Rich Internet Applications) — набор атрибутов для улучшения доступности.

```html
<!-- Роль элемента -->
<div role="navigation">...</div>

<!-- Подпись для скринридеров -->
<button aria-label="Закрыть">×</button>

<!-- Описание связи -->
<input aria-describedby="password-hint">
<p id="password-hint">Минимум 8 символов</p>

<!-- Состояние -->
<button aria-expanded="false">Подробнее</button>
```

### Правила доступности

1. **Все изображения должны иметь alt**
2. **Формы должны иметь label**
3. **Ссылки должны быть понятными** (не "кликните здесь")
4. **Используйте семантические теги**
5. **Обеспечьте навигацию с клавиатуры**
