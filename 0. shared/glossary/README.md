A comprehensive glossary of terms used throughout the course.

## A

**API (Application Programming Interface)**  
A set of rules and protocols that allows different software applications to communicate with each other.

**Accessibility (a11y)**  
The practice of making websites usable by people of all abilities and disabilities.

**AJAX (Asynchronous JavaScript and XML)**  
A technique for creating dynamic web pages by exchanging data with a server asynchronously.

**Array**  
A data structure that stores multiple values in a single variable.

## B

**Backend**  
The server-side of a web application that handles data processing, storage, and business logic.

**Box Model**  
The CSS concept that describes how elements are rendered with content, padding, border, and margin.

**Browser**  
Software application used to access and view websites (e.g., Chrome, Firefox, Safari).

## C

**CSS (Cascading Style Sheets)**  
A stylesheet language used to describe the presentation of HTML documents.

**CRUD**  
Create, Read, Update, Delete - the four basic operations for persistent storage.

**Client**  
The user's device or browser that requests and displays web content.

**Callback**  
A function passed as an argument to another function to be executed later.

## D

**Database**  
An organized collection of structured data stored electronically.

**DBMS (Database Management System)**  
Software that manages databases (e.g., MySQL, PostgreSQL).

**DOM (Document Object Model)**  
A programming interface for HTML documents that represents the page structure as a tree of objects.

**Deployment**  
The process of making a web application available on a server for users to access.

## E

**Element**  
A component of an HTML document defined by tags (e.g., `<div>`, `<p>`, `<h1>`).

**Event**  
An action or occurrence detected by the browser (e.g., click, hover, submit).

**Express.js**  
A minimal and flexible Node.js web application framework.

## F

**Frontend**  
The client-side of a web application that users interact with directly.

**Flexbox**  
A CSS layout module for creating flexible and responsive layouts.

**Function**  
A reusable block of code that performs a specific task.

## G

**Git**  
A distributed version control system for tracking changes in source code.

**Grid**  
A CSS layout system for creating two-dimensional layouts with rows and columns.

## H

**HTML (HyperText Markup Language)**  
The standard markup language for creating web pages.

**HTTP (HyperText Transfer Protocol)**  
The protocol used for transmitting data over the web.

**HTTPS**  
HTTP Secure - encrypted version of HTTP using SSL/TLS.

## I

**IDE (Integrated Development Environment)**  
Software for writing, testing, and debugging code (e.g., VS Code).

**IP Address**  
A unique numerical identifier for devices on a network.

## J

**JavaScript**  
A programming language that enables interactive web pages.

**JSON (JavaScript Object Notation)**  
A lightweight data interchange format.

**JOIN**  
SQL operation that combines rows from two or more tables.

## K

**Key**  
In databases, a field used to uniquely identify records (primary key) or establish relationships (foreign key).

## L

**Layout**  
The arrangement of visual elements on a web page.

**Library**  
A collection of pre-written code for common tasks.

**Localhost**  
The local computer used as a server for development (usually 127.0.0.1).

## M

**Middleware**  
Functions that execute during the request-response cycle in Express.js.

**MySQL**  
An open-source relational database management system.

**MVC (Model-View-Controller)**  
An architectural pattern that separates application logic into three components.

## N

**Node.js**  
A JavaScript runtime built on Chrome's V8 engine for server-side programming.

**npm (Node Package Manager)**  
The default package manager for Node.js.

**Normalization**  
The process of organizing database tables to reduce redundancy.

## O

**Object**  
A data structure that stores key-value pairs.

**ORM (Object-Relational Mapping)**  
A technique for converting data between incompatible type systems.

## P

**Package**  
A bundle of code that can be installed and used in projects.

**Parameter**  
A variable in a function definition; values passed to functions are arguments.

**Promise**  
An object representing the eventual completion or failure of an asynchronous operation.

**Primary Key**  
A unique identifier for each record in a database table.

## Q

**Query**  
A request for data from a database.

**Query String**  
Parameters appended to a URL (e.g., `?name=value&age=25`).

## R

**REST (Representational State Transfer)**  
An architectural style for designing networked applications using HTTP.

**Responsive Design**  
Designing websites that adapt to different screen sizes and devices.

**Route**  
A URL pattern that maps to specific application logic.

## S

**Semantic HTML**  
Using HTML elements that clearly describe their meaning (e.g., `<article>`, `<nav>`).

**Server**  
A computer or program that provides services to clients over a network.

**SQL (Structured Query Language)**  
A language for managing and querying relational databases.

**Selector**  
A pattern used to select HTML elements for styling with CSS.

## T

**Table**  
A database structure that organizes data in rows and columns.

**Template**  
A pre-designed file or layout used as a starting point.

**Token**  
A piece of data used for authentication and authorization.

## U

**URL (Uniform Resource Locator)**  
The address of a resource on the web.

**UI (User Interface)**  
The visual elements users interact with.

**UX (User Experience)**  
The overall experience of using a product or service.

## V

**Variable**  
A named storage location for data in programming.

**Version Control**  
A system for tracking changes to files over time (e.g., Git).

**View**  
In MVC, the presentation layer that displays data to users.

## W

**Web Server**  
Software that serves web pages to clients (e.g., Apache, Nginx).

**Webpack**  
A module bundler for JavaScript applications.

## X

**XML (eXtensible Markup Language)**  
A markup language for encoding documents in a format that is both human and machine-readable.

## Y

**YAML**  
A human-readable data serialization format.

## Z

**Zero-based Indexing**  
A numbering system where the first element has index 0.
