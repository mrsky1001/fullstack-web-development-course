## 📚 Теоретическая часть

### Что такое данные?

**Данные** — это информация в структурированном виде: числа, текст, даты, изображения.

Примеры данных:
- Список пользователей сайта
- Товары интернет-магазина
- Посты в социальной сети
- Транзакции банка

### Способы хранения данных

| Способ | Плюсы | Минусы |
|--------|-------|--------|
| **Переменные** | Быстро | Пропадают при перезапуске |
| **Файлы** | Простота | Медленно, сложно искать |
| **База данных** | Надёжно, быстро, структурировано | Требует настройки |

### Пример: Хранение пользователей

**В файле (JSON):**
```json
[
    { "id": 1, "name": "Иван", "email": "ivan@mail.ru" },
    { "id": 2, "name": "Мария", "email": "maria@mail.ru" }
]
```

Проблемы:
- Как найти пользователя по email?
- Как обновить данные одного пользователя?
- Что если файл повредится?
- Что если 2 процесса пишут одновременно?

**В базе данных:**
```sql
SELECT * FROM users WHERE email = 'ivan@mail.ru';
```

Преимущества:
- Быстрый поиск по любому полю
- Транзакции и целостность данных
- Одновременный доступ множества пользователей
- Резервное копирование и восстановление

## 🏗️ Архитектура веб-приложения

```
┌─────────────┐      ┌─────────────┐      ┌─────────────┐
│   Frontend  │ ←──→ │   Backend   │ ←──→ │  Database   │
│  (Браузер)  │      │  (Сервер)   │      │   (MySQL)   │
└─────────────┘      └─────────────┘      └─────────────┘
     HTML/JS            Node.js              SQL
   Отображение        Бизнес-логика       Хранение
```

### Как это работает?

1. Пользователь открывает страницу товаров
2. Frontend отправляет запрос на Backend
3. Backend делает SQL-запрос к базе данных
4. База данных возвращает данные
5. Backend формирует ответ (JSON)
6. Frontend отображает товары

```
Пользователь:  "Покажи товары"
     ↓
Frontend:      GET /api/products
     ↓
Backend:       SELECT * FROM products
     ↓
Database:      [ { id: 1, name: "iPhone", price: 99990 }, ... ]
     ↓
Frontend:      Отображает карточки товаров
```

## 💾 Типы баз данных

### Реляционные (SQL)

- Данные хранятся в **таблицах**
- Таблицы связаны между собой
- Язык запросов: **SQL**
- Примеры: MySQL, PostgreSQL, SQLite, Oracle

```
Таблица "users":
┌────┬────────┬─────────────────┐
│ id │ name   │ email           │
├────┼────────┼─────────────────┤
│ 1  │ Иван   │ ivan@mail.ru    │
│ 2  │ Мария  │ maria@mail.ru   │
└────┴────────┴─────────────────┘
```

### Нереляционные (NoSQL)

- Данные хранятся в **документах**, **ключ-значение**, **графах**
- Более гибкая структура
- Примеры: MongoDB, Redis, Cassandra

```javascript
// MongoDB (документ)
{
    _id: "abc123",
    name: "Иван",
    email: "ivan@mail.ru",
    orders: [
        { product: "iPhone", date: "2024-01-15" }
    ]
}
```

### Когда что использовать?

| Тип | Когда использовать |
|-----|-------------------|
| **SQL (MySQL)** | Структурированные данные, связи между сущностями |
| **NoSQL (MongoDB)** | Гибкая структура, большие объёмы, JSON-документы |
| **Redis** | Кэширование, сессии, очереди |

**В этом курсе мы изучаем SQL и MySQL** — это основа для большинства приложений.

## 🛠️ Инструменты курса

### MySQL Server

**MySQL** — бесплатная реляционная СУБД. Одна из самых популярных.

- Разработчик: Oracle (ранее MySQL AB)
- Версия: 8.x
- Лицензия: GPL (бесплатно)

### MySQL Workbench

**MySQL Workbench** — графический интерфейс для работы с MySQL.

Возможности:
- Выполнение SQL-запросов
- Визуальное проектирование БД
- Импорт/экспорт данных
- Администрирование сервера

### Альтернативные инструменты

| Инструмент | Описание |
|------------|----------|
| **DBeaver** | Универсальный клиент для любых БД |
| **HeidiSQL** | Лёгкий клиент для Windows |
| **phpMyAdmin** | Веб-интерфейс для MySQL |
| **DataGrip** | IDE от JetBrains (платная) |
| **VS Code + SQLTools** | Расширение для VS Code |

## 📖 Терминология

| Термин | Описание | Аналогия |
|--------|----------|----------|
| **Database** | База данных | Папка с файлами |
| **Table** | Таблица | Excel-таблица |
| **Row** (запись) | Строка таблицы | Одна строка в Excel |
| **Column** (поле) | Столбец таблицы | Один столбец в Excel |
| **Primary Key** | Уникальный ID | Номер паспорта |
| **Foreign Key** | Ссылка на другую таблицу | Ссылка на документ |
| **Query** | Запрос | Вопрос к базе данных |
| **CRUD** | Create, Read, Update, Delete | Основные операции |
